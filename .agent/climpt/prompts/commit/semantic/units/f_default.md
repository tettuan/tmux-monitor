---
title: 意味的近さでコミットを分けて実施する
---

# 指示
Gitコミットを、ファイルの変更内容の近さ単位でグループ化し、
近い単位にまとめてコミットする。

まったく異なる内容を1つのコミットへ含めることを避けつつ、
複数回のコミット処理を連続して実行することが目的である。

意味的に近いとは、

ある変更について、同じ変更に関与した
- documents
- prompts
- codes
- tests
などのファイル種類を1つのコミットにまとめることである。

一方、複数のプロンプトやcodeでも、意味的に異なる変更は分けてコミットすることである。

# 手順

- git status
- 変更を分類をする

2つに分けられる場合:
- git add 1
- git commit 1
- git add 2
- git commit 2
